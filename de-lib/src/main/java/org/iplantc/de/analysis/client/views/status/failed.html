My analysis failed

There are several common causes for a failed analysis.

When there is scheduled DE maintenance, all submitted analyses will be killed. If your analysis fails on a maintenance day, please relaunch the analysis.

Problem with the input file(s):

* Special characters in the file name: Rename the file, excluding any of the special characters and relaunch ananlysis.

* Incorrect input file format: Read the app documentation and reformat input file. Relaunch analysis.

* Corrupted input file: Try to run the app again using another input file. Start with the example file supplied with the app. You may need to return to your original data source (e.g., sequencing center) and request a new copy of the input data.

Other common reasons for analysis failure. Solution to the issue depends on the status of the app (public, beta, or private):

* Invalid parameters: Each tool will report errors differently. You will need to search the log files for an error message. Start by searching the (stdout) file, which lists all of the output from the tool. Also check the app documentation, including the original documentation for the tool.

* Problem with app or with the source code. Which of the following applies?

* The app is public and no longer in beta: A problem with the code is unlikely for a public, non-beta app. Please post a comment.

* The app is public but still in beta: If it is not your app, and it is in the beta category, please post a comment.

* The app is private: If it is your app, be sure to validate with a real dataset or two, not just the example data that comes with the code. There may be a code bug in the executable that drives the app. Check the discussion list or issue tracker for the tool, and email the list or file a ticket if necessary. CyVerse staff cannot help with troubleshooting externally generated code.
